# 「搭把手」To G端 · 社区运营管理后台


## 📖 项目简介

**「搭把手」To G端是社区运营管理后台**，为街道办、社区/居委会、党群服务中心、妇联等G端用户提供数据看板、内容管理和需求处理工具。

### 核心定位

> **社区运营的“驾驶舱”，社区治理的“数据底座”。**

### 为什么需要 To G 端

「搭把手」To C端（微信小程序）是居民每天使用的产品，产生了大量行为数据——活动报名、公告查看、邻里互助、关怀打卡、诉求反馈等。但社区运营人员需要一个后台来**查看这些数据、管理这些内容、回应这些需求**。

**核心原则**：所有数据来自居民端真实行为，G端无需额外录入。不增加基层负担。


## 🎯 目标用户

| 角色 | 权限范围 | 核心需求 |
|---|---|---|
| **街道办工作人员** | 全域数据、导出报告 | 看数据、做汇报、发现辖区问题 |
| **社区/居委会工作人员** | 本社区内容+数据 | 发布活动/公告、查看诉求、服务居民 |
| **党群服务中心运营人员** | 本中心活动+阵地数据 | 提升空间使用率、组织党群活动 |
| **妇联/群团组织工作人员** | 帮扶+特殊群体数据 | 触达特殊群体、派单给志愿者 |

**登录方式**：手机号+验证码登录，登录后下拉选择身份。


## ✨ 核心功能

### 1. 工作台首页

登录后默认进入，一屏掌握全局：

| 区域 | 内容 |
|---|---|
| **数据速览卡片** | 6项核心指标：注册居民、本周活跃、活动参与率、互助完成率、诉求闭环率、居民满意度 |
| **今日待办** | 需处理事项列表（活动报名超预期/待审核/诉求待回复/帮扶待派单/未报平安提醒），点击直接跳转 |
| **快捷操作** | 4个核心按钮：发布活动、发布公告、查看诉求、查看完整数据 |

### 2. 数据看板

六大板块 + 自动周报：

| 板块 | 数据来源（ToC功能） | 展示内容 |
|---|---|---|
| 居民活跃度 | 用户登录/使用记录 | 日活趋势图、功能使用分布 |
| 活动参与数据 | 活动报名/签到记录 | 活动发布数/报名数/签到率/最受欢迎活动TOP5 |
| 诉求分析数据 | 反馈问题记录 | 诉求类型分布/高频关键词TOP5/趋势图 |
| 互助完成数据 | 邻里圈互助记录 | 互助发布量/响应量/完成率/平均响应时间 |
| 关怀版专项数据 | 关怀版使用数据 | 报平安人次/帮扶申请量/用药打卡率/一键呼救数据 |
| 社区治理总览 | 以上数据聚合 | 本周总览/各社区排名（街道视角） |

**周报/月报**：自动生成，一键导出PDF，直接用于向上汇报。

### 3. 自治管理控制台

基于ToC功能的管理操作，**一处发布、多处生效**：

| 模块 | 管理内容 | 对应ToC功能 |
|---|---|---|
| 活动管理 | 发布/审核/报名/签到 | 社区地图→本周活动 |
| 公告管理 | 发布/分类/已读追踪 | 首页→最新公告 |
| 空间管理 | 公共空间信息维护 | 社区地图→公共空间 |
| 服务管理 | 周边服务信息维护 | 社区地图→周边服务 |
| 社工管理 | 社工信息维护 | 社区地图→社工服务 |
| 居民管理 | 居民档案/标签/特殊群体标记 | 我的→个人信息 |
| 防诈骗管理 | 每日提醒内容配置 | 首页→防诈骗提醒（关怀版） |

### 4. 需求反馈

当居民遇到AI问答无法解决的问题时，提供的兜底通道：

| 通道 | 适用场景 | G端操作 |
|---|---|---|
| 💬 留言板 | 非紧急问题/建议/政策咨询 | 查看→回复→标记状态（待处理/处理中/已解决） |
| 📞 电话热线 | 紧急问题（停水/爆管/突发状况） | 电话列表配置/通话记录 |

## 🚀 快速开始

### 前置条件

- Node.js 18+
- MySQL 8.0+（如需本地运行完整版）
- Redis 7.0+（如需本地运行完整版）

### 1. 克隆项目

```bash
git clone https://github.com/yourusername/dabashou-admin.git
cd dabashou-admin
```

### 2. 安装依赖

```bash
npm install
```

### 3. 配置环境变量

```bash
cp .env.example .env
# 编辑 .env 配置 API 地址
```

### 4. 启动开发服务

```bash
npm run dev
```

访问 `http://localhost:5173` 预览。

### 5. 构建生产版本

```bash
npm run build
```

## 🎨 设计原则

| 原则 | 说明 |
|---|---|
| **轻量** | 不增加基层负担，不用天天登录，需要时打开就能用 |
| **全面不复杂** | 覆盖ToC所有功能的管理需求，但每个功能入口清晰、路径单一 |
| **数据驱动** | 所有数据来自居民端真实行为，G端无需额外录入 |
| **好用** | 操作路径3步以内，关键操作一键直达 |
| **一处发布、多处生效** | 管理后台所有操作结果实时同步到居民端小程序 |
